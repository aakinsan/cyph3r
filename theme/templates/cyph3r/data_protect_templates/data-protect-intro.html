Data Encryption/Decryption

The application makes use of 2 AES modes - AES GCM and AES CBC for encryption and decryption operations. 128, 192 and 256 bit key sizes are supported.

Both modes require the use of an Initialization Vector (IV or alternatively a nonce)

AES GCM

This app only supports splitting 128 bit keys using Shamir's secret sharing. The key can be broken into n shares and a minimum of m shares (m < n) otherwise known as the threshold is required for key reconstruction.

Xor-based Key Splitting

A much simpler approach than Shamir is using the XOR or bitwise operation to split a secret. All the shares are required to reconstruct the secret.

The assumption is the key shares are also expected to remain secret. Consequently, every key splitting operation requires PGP public files to be uploaded. The number of files to be uploaded must be the same number as that of the shares to be created.

Key reconstruction operations require only a single PGP file to be uploaded.

This app only accepts PGP public key files that are in the ASCII armored format i.e. the contents of each public key file must have the begin and end headers of ----- BEGIN PGP PUBLIC KEY BLOCK----- and -----END PGP PUBLIC KEY BLOCK----- respectively.
