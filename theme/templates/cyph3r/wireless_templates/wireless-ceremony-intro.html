Key Ceremony

Objective

Secure generation and protection of four wireless keys:

- Tuak TOP and Transport Keys

- Milenage OP and Transport Keys

Requirements

SIM Provider Public Keys

3 PGP public keys for the SIM provider are required to encrypt the 3 XOR key shares generated for each wireless keys. The provider will reconstruct each key after decypting the shares with their private keys.

Security Officer Public Keys

A unique 128-bit wrap key is generated for each wireless key. The wrap key is used to wrap/encrypt its associated wireless key. Each wrap key is then split into 5 shares using Shamir Secret Splitting (SSS). Each of these shares will be encrypted with a PGP public key belonging to a security officer. Three security officers will be required to reconstruct this wrap key which in turn can be used to decrypt its wireless key.

Public Keys for the Milenage

The milenage key will be entered at the terminal of the network provider. To mitigate the risk of both milenage keys been known by a single resource, two public keys belonging to two different resources will be provided. One to encrypt the milenage Transport key and the other to encrypt the milenage OP key.

This app only accepts PGP public key files that are in the ASCII Armored format i.e. the contents of each public key file must have the begin and end headers of ----- BEGIN PGP PUBLIC KEY BLOCK----- and -----END PGP PUBLIC KEY BLOCK----- respectively.

Finally, a GCP project ID can be specified to store the wrapped secrets in a secret manager vault. This is optional since all keys will be encrypted and available for download at the end of this process.
